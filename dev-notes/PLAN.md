# Agent · 需求计划

> 主入口 `scipi/`。目标：上下文更简洁、对长任务支持更好的 agent。

## 需求 1 — 精简工具与提示

**问题**：现 32 个工具太多，system prompt 太杂，白白吃上下文。

**做法**：
- 大删不常用工具，只留高频核心集
- 精简 system prompt，去冗余、缩篇幅

**收益**：上下文更简洁。

## 需求 2 — 先读项目再动手

**问题**：agent 不读项目就直接改，体验差、易出错。

**做法**：
- 入口强制探索：先读项目结构与关键上下文
- 未读上下文前，禁止直接编辑

**收益**：长任务更稳、体验更好。
