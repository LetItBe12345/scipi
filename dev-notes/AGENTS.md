# 工作区说明

主仓库：`scipi/`（Git 仓库根位于 `scipi/.git`）

目标：构建一个更简洁、上下文更精简、对长任务支持更好的 Agent。

## 工作要求

- **回答言简意赅**：内容要短，不要长篇大论。
- **输出要给人看**：多分段、短句、简单词；先结论后细节，控制篇幅。
- **能少改就少改**：写代码尽量小步、最小改动。
- **能复用就复用**：优先用已有代码和现成实现。
- **先搜索再动手**：互联网有现成方案就用，站在巨人肩膀上，不要凡事自己实现。
- **先阅读再修改**：任何行为，包括讲解代码和修改代码时，都要先调用工具阅读提示用户需求指明的相关的部分后再回答，不要幻觉，一定要将证据落实。
- **文档归档到位**：分析文档、调研笔记等放到 `/home/jin/pi/docs/`，别乱放。
- **改完即提交**：在主仓库 `scipi/` 进行任意一次代码修改后，都要进行一次 `git commit`。
- **提交信息写详细**：`git commit` 的 message 要写清这次具体干了什么，别笼统。
